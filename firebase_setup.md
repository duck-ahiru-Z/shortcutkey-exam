# Firebase / Firestore セットアップ手順書

本システムで受験ログの保存やダッシュボードでの統計情報の閲覧を行うために、以下の手順で Firebase（Cloud Firestore）をセットアップしてください。

---

## ステップ 1. Firebase プロジェクトの作成

1. **[Firebase Console](https://console.firebase.google.com/)** にブラウザでアクセスし、お使いの Google アカウントでログインします。
2. **「プロジェクトを追加」** または **「プロジェクトを作成」** ボタンをクリックします。
3. プロジェクト名（例: `shortcut-key-exam`）を入力し、規約に同意して「続行」をクリックします。
4. Google アナリティクスを設定するかどうかの選択肢はお好みに応じて有効/無効にし、「プロジェクトを作成」をクリックします（無効で問題ありません）。
5. 数十秒待つとプロジェクトが作成されます。「続行」をクリックしてダッシュボードを開きます。

---

## ステップ 2. Cloud Firestore データベースの有効化

1. 左メニューの「構築」グループから **「Firestore Database」** を選択します。
2. 画面中央の **「データベースの作成」** ボタンをクリックします。
3. ロケーション（保存場所）を選択します。アジア地域であれば **`asia-northeast1` (東京)** または **`asia-northeast2` (大阪)** を選択することをおすすめします。「次へ」をクリックします。
4. 開発中に限り **「テストモードで開始する」** を選択し、「作成」をクリックします。
   - テストモードでは一時的に第三者からデータを読み書きされる可能性があるため、公開運用には使用しないでください。
   - `allow read, write: if true;` のように全データを無条件で許可するルールは、本番環境では設定しないでください。

### 公開運用前に必要なセキュリティ設定

現在のクライアント実装だけでは、匿名受験者による記録追加と管理者だけの統計閲覧を安全に両立できません。公開運用前に、少なくとも以下を実施してください。

1. Firebase Authenticationを使って管理者を認証する
2. `exam_records`への新規記録だけを必要なフィールドに限定して許可する
3. 成績の読み取り・更新・削除は認証済み管理者だけに許可する
4. Firebase App Checkを有効にし、不正なクライアントからの送信を抑制する
5. Rules SimulatorまたはFirebase Emulatorで拒否・許可の両方をテストしてからルールを公開する

管理者認証とSecurity Rulesが整うまでは、`dashboard.html`はlocalhost上での開発確認にのみ使用してください。

---

## ステップ 3. Webアプリの登録と設定情報の取得

1. Firebase Console のホーム画面（左上メニューの「プロジェクトの概要」をクリック）に戻ります。
2. アプリの追加アイコンの中から **「ウェブ」アイコン（`</>`）** をクリックします。
3. アプリのニックネーム（例: `ibt-app`）を入力します。
   - 「このアプリの Firebase Hosting も設定します」のチェックは不要です。
4. **「アプリを登録」** をクリックします。
5. 画面に設定コードが表示されます。コード内の `const firebaseConfig = { ... };` の中身をコピーします。

---

## ステップ 4. システムへの設定情報の流し込み

1. 本プロジェクト内にある **`firebase_config.js`** ファイルをエディタで開きます。
2. コピーした Firebase の設定オブジェクト（`apiKey` や `projectId` など）を、以下の該当箇所へそのまま貼り付けて上書き保存します。
   ```javascript
   // firebase_config.js
   const firebaseConfig = {
     apiKey: "貼り付けたAPIキー",
     authDomain: "貼り付けたauthDomain",
     projectId: "貼り付けたprojectId",
     // ... 残りのパラメータも同様に貼り付け
   };
   ```

---

## 完了

設定完了後、ブラウザで受験を行って合格または終了すると、Firestore 内に自動的に `exam_records` コレクションが作成され、受験情報がリアルタイムに保存されるようになります！

設定が完了するまでの間は、ブラウザの `localStorage` を使用した**「デモ/モックモード」**で自動稼働し、ダッシュボードの表示やデータ送信の流れをテストできます。
